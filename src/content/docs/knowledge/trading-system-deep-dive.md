# 交易系统基础深度笔记

## 1. 学习目标

交易系统经历是 C++ 工程能力的辅助证明，不应包装成金融核心系统专家。重点是能讲清行情、委托、撮合、订单状态流转和单元测试。

## 2. 基本模块

| 模块 | 作用 |
|---|---|
| 行情模块 | 接收、解析、缓存和分发市场数据 |
| 委托模块 | 接收策略订单、校验、发送、维护订单状态 |
| 撮合模块 | 根据规则生成成交结果，可用于回测或模拟交易 |
| 策略接入 | 将行情、账户、订单、成交回报暴露给策略 |
| 风控模块 | 检查资金、仓位、价格、频率等约束 |

## 3. 订单生命周期

```text
New
  -> PendingSubmit
  -> Submitted
  -> PartiallyFilled
  -> Filled
  -> CancelPending
  -> Cancelled
  -> Rejected
```

需要注意：真实系统中订单状态可能乱序到达、重复到达或部分成交后撤单，所以状态机必须有明确合法转移。

## 4. 撮合规则

常见规则是价格优先、时间优先：

- 买单价格越高优先级越高；
- 卖单价格越低优先级越高；
- 同价格下先到先成交。

撮合模块需要处理限价单、市价单、部分成交、撤单、盘口更新等情况。

## 5. 行情模块关注点

- 数据解析和字段校验；
- 时间戳和顺序；
- 快照与增量更新；
- 丢包或乱序处理；
- 行情分发给策略的延迟；
- 数据结构设计，例如 order book、ring buffer、队列。

## 6. 委托模块关注点

- 订单参数校验；
- 状态机维护；
- 订单 ID 映射；
- 发送失败、拒单、撤单、部分成交；
- 成交回报和策略侧状态同步。

## 7. 单元测试方向

- 新订单正常流转；
- 拒单；
- 部分成交；
- 完全成交；
- 撤单成功；
- 撤单失败；
- 重复回报；
- 非法状态转移；
- 行情字段解析；
- 撮合价格时间优先。

## 8. 面试边界

可以说：参与行情、委托、撮合模块开发和测试，理解交易链路和订单状态流转。

谨慎说：负责整个交易系统架构或低延迟核心撮合引擎。实习经历要保持边界。

## 9. 高频追问

- 行情、委托、撮合分别负责什么？
- 一个订单从策略发出到成交回报经历哪些状态？
- 限价单和市价单区别是什么？
- 撮合中的价格优先、时间优先怎么实现？
- 如何测试订单状态机？
- 如果成交回报重复到达怎么办？
